# 宣传视频创作工作流 (Promo Video Creator)

## 核心原则

本技能提供一套完整的宣传视频创作方法论，覆盖从素材分析到最终视频生成的全流程。适用于企业宣传片、产品介绍视频、服务展示视频等场景。

**语言策略（择优选择）：**
- **字幕/旁白**：始终使用中文简体，面向中国受众
- **分镜图提示词**：默认使用中文（中国场景描述更精准，现代AI模型对中文支持良好）；如目标生成工具对英文响应更优，可切换英文
- **分镜视频提示词**：默认使用中文；包含动态描述、运镜方式、转场效果
- **品牌名称/专业术语**：保持原文，必要时中英对照

**视频生成策略：**
- 每个分镜先生成分镜参考图（关键帧），再以「分镜参考图 + 分镜视频提示词」共同驱动视频生成
- 图片生成使用 Agnes Image 2.1 Flash 或用户指定的图片生成工具
- 视频生成使用 Agnes Video V2.0（image-to-video模式）或用户指定的视频生成工具

**中间产物全量保留原则（⚠️ 核心原则）：**
- **所有阶段的生成内容必须完整保留，不得删除或覆盖。** 这包括但不限于：
  - 各阶段设计文件（计划方案、字幕文稿、分镜总设计）
  - 所有分镜提示词文件（含图提示词、视频提示词、字幕）
  - 所有生成的分镜关键帧图片（即使质量不满意也保留，标注为备选）
  - 所有生成的分镜视频片段（逐个保存，不合并删除）
- **保留目的**：若最终AI生成的视频效果不理想，用户可基于所有中间产物自行使用专业剪辑软件（如 Premiere、DaVinci Resolve、剪映等）进行手动剪辑合成
- **最终输出**：HyperFrames 自动拼接的完整视频 + 全部独立分镜视频 + 全部中间文件，用户可根据需要选择使用

#### 检查 C：Windows 中文编码兼容性（⚠️ Windows 用户必检）

**在 Windows 上，命令行传入的中文提示词会被系统编码（GBK/CP936）错误解码，导致 API 收到乱码，生成的图片/视频与提示词完全无关。**

**症状**：提示词正确，但生成的图片/视频内容完全不对（不是风格或细节问题，而是根本不同）。

**检查方式**：执行 dry-run 测试

```powershell
# 测试中文是否能正确传输
PYTHONUTF8=1 python scripts/agnes_api.py image --prompt "测试中文红色苹果" --size 256x256 --dry-run
```

查看输出中 `"prompt"` 字段的值——如果显示为乱码（如 `һֻ`），说明编码有问题。

**解决方案（三选一）**：

| 方案 | 用法 | 适用场景 |
|------|------|---------|
| **① 环境变量前缀**（推荐） | `PYTHONUTF8=1 python scripts/agnes_api.py ...` | 所有场景 |
| **② 文件读入** | `--prompt-file prompt.txt`（先用 Python 写 UTF-8 文件） | 超长提示词 |
| **③ 包装脚本** | `./agnes.sh` 或 `agnes.bat`（自动设置 PYTHONUTF8=1） | 频繁使用 |

**⚠️ 关键规则：每次调用 agnes_api.py 时必须在命令前加 `PYTHONUTF8=1`。在 Subagent 的 prompt 中也要明确要求这一点。**

#### 检查 D：API Key 是否有效（可选快速验证）

如果用户即将进入图片/视频生成阶段（第五、六阶段），建议执行快速连通性验证：

```powershell
PYTHONUTF8=1 python scripts/agnes_api.py image \
  --prompt "test" \
  --size 256x256 \
  --dry-run
```

如果返回认证错误（401/403），说明 API Key 无效或已过期，需重新配置。

## 第一阶段：需求收集与素材分析

### 1.1 任务需求四问（⚠️ 必须询问 — 不可跳过）

**在开始任何创作工作之前，必须向用户逐一确认以下四个维度。这是理解创作方向的根基，不得以任何理由跳过或简化。**

```

  · 是否有项目说明文档或产品介绍资料？（.docx / .pdf / .pptx / 网页链接）
  · 是否有可以参考的现有宣传视频？（本地文件 / 在线链接）

  · 是否有品牌视觉规范？（LOGO文件 / 配色方案 / 字体要求 / VI手册）
```

### 1.2 参考素材收集与整理

对每个参考视频，分析并记录：
- **总时长与字幕条数**：计算平均每条字幕的展示时长（通常2.5-4秒）
- **段落结构**：识别视频分为几个段落（痛点→方案→功能→系统→流程→价值→背书→展望）
- **各段落时长占比**：统计每段占总时长的百分比
- **转场手法**：记录段落之间的转场方式
- **视觉风格**：色调、运镜、动画风格
- **字幕语言风格**：正式度、句子长度、信息密度

**标准宣传片结构模式（从参考视频归纳）：**
```
[痛点共鸣 10-15%] → [方案引出 6-12%] → [产品功能 14-18%] → [系统联动 14-16%]
→ [运营流程 18-24%] → [价值总结 10-18%] → [案例背书 8-13%] → [未来展望 5-8%]
```

### 1.3 目标场景信息提取

从目标场景文档中系统提取：
- **痛点/背景**：当前面临的什么问题需要解决？
- **政策/行业驱动**：有什么政策或行业要求推动变革？
- **解决方案**：产品/服务是什么？核心能力有哪些？
- **核心功能**：具体的功能点列表
- **运营保障**：如何保障服务持续运行？
- **核心价值**：带来的关键收益（量化优先）
- **落地案例**：已在哪些客户/场景使用？
- **资质背书**：有什么认证、保险、安全资质？

## 第二阶段：方案设计

### 2.1 创建整体计划方案文件

生成 `宣传视频创作计划方案.md`，包含：
1. 项目概述（目标、素材来源、核心信息提炼）
2. 参考视频结构深度分析（逐段拆解）
3. 目标视频创作方案（基本参数、分镜总数、时长分配）

## 第七阶段：HyperFrames 视频拼接合成（⚠️ 第三方外部 Skill）

### 7.1 HyperFrames Skill 最终确认

**HyperFrames 是一个功能丰富的视频处理 Skill，本阶段主要使用其拼接和转场能力，但你可以在后续步骤中利用它的更多功能（如特效叠加、色彩统一、多轨道合成等）来进一步提升视频质量。**

**在进入本阶段之前，再次确认 HyperFrames 可用性：**

- 在第零阶段已预检并尝试安装，此处做最终确认
- 若 HyperFrames 不可用：第四、五、六阶段生成的所有分镜视频和中间文件均已保留，用户可随时使用 HyperFrames 或手动剪辑（参见「手动剪辑备选方案」）
- 若 HyperFrames 已安装：直接进入拼接流程，并可利用其更多高级功能

### 7.2 拼接合成要求

**核心目标：将 N 个分镜视频按序号拼接为一部完整宣传视频，每两个视频之间添加自然转场。**

```
输入：15个分镜视频文件（按分镜序号命名）
  ├── 分镜01_视频.mp4（12秒）
  ├── 分镜02_视频.mp4（14秒）
  ├── ...
  └── 分镜15_视频.mp4（14秒）

拼接流程（HyperFrames 处理）：
  分镜01 ──[转场]──▶ 分镜02 ──[转场]──▶ 分镜03 ──[...]──▶ 分镜15
  
输出：
  └── 最终成片.mp4（~180秒，1920×1080，24fps）
```

### 7.3 转场设计原则

**每两个分镜之间必须有一个自然的转场过渡，根据分镜的情绪和视觉特点选择合适的转场类型：**

| 转场节点 | 分镜间情绪变化 | 推荐转场类型 | 转场时长 |
|---------|-------------|-------------|:---:|
| 01→02 | 纪实→沉重 | 叠化（Dissolve）+ 色调变暗 | 0.8s |
| 02→03 | 沉重→权威 | 闪白切（Flash White Cut） | 0.5s |
| 03→04 | 权威→希望 | 光效过渡（Light Sweep） | 1.0s |
| 04→05 | 品牌→产品 | 聚焦推进（Rack Focus Push） | 0.8s |
| 05→06 | 外观→功能 | 匹配剪辑（Match Cut） | 0.6s |
| 06→07 | 清点→缴存 | 连续动作剪辑（Cut-on-Action） | 0.5s |
| 07→08 | 物理→数字 | 数据化过渡（Data Morph） | 1.0s |
| 08→09 | 同步→监控 | 拉远展开（Zoom Out） | 0.8s |
| 09→10 | 监控→结算 | UI界面推进（UI Zoom-In） | 0.7s |
| 10→11 | 资金→运营 | 擦除过渡（Wipe） | 0.6s |
| 11→12 | 操作→闭环 | 图形匹配（Shape Match） | 0.8s |
| 12→13 | 闭环→价值 | 粒子爆发（Particle Burst） | 1.0s |
| 13→14 | 价值→案例 | 地图定位（Map Pin） | 0.8s |
| 14→15 | 案例→展望 | 远景拉升（Aerial Pull-Back） | 1.2s |
| 15→结尾 | 展望→定格 | 渐黑（Fade to Black）+ LOGO | 1.5s |

### 7.4 HyperFrames 调用方式

**在确认 HyperFrames Skill 已安装后，通过 Skill 工具调用：**

```
使用 HyperFrames Skill 进行以下视频合成：

【输入视频列表】（按拼接顺序）
1. output/storyboards/videos/分镜01_视频.mp4
2. output/storyboards/videos/分镜02_视频.mp4
...
15. output/storyboards/videos/分镜15_视频.mp4

【转场要求】
- 每两个视频之间使用上述转场类型
- 转场时长按上表设定
- 保持画面16:9比例和1920×1080分辨率
- 帧率统一为24fps

【输出】
- 文件路径：output/最终成片.mp4
- 总时长：约180秒
```

### 7.5 后期合成补充步骤

HyperFrames 完成视频拼接后，还需进行以下补充处理：

1. **字幕叠加**：根据各分镜的字幕内容和时间码，在对应时间位置叠加字幕轨道
2. **音频处理**：添加背景音乐（BGM）和配音旁白（VO）
3. **色彩统一**：检查全片色调一致性，必要时进行色彩校正
4. **最终输出**：1920×1080, 24fps, H.264编码, 比特率 ≥ 15Mbps

## 手动剪辑备选方案（⚠️ 用户自助指南）

### 为什么需要手动剪辑？

AI 生成的视频可能存在以下局限：
- 画面动态不够自然，人物动作或表情有瑕疵
- 转场过渡不够平滑
- 色调或光影在分镜间有轻微不一致
- 视频提示词的动态效果未完全实现
- 某些分镜生成的视频质量明显低于其他分镜

**因此，Skill 保留全部中间产物，让用户可以：**
1. 挑选质量满意的分镜片段直接使用
2. 对不满意的分镜使用专业工具重新生成或手动调整
3. 自行控制转场类型和时长
4. 添加配音、BGM、字幕等后期元素

### 手动剪辑工作流

```
准备工作：
  1. 打开 output/storyboards/videos/ 目录，预览15个分镜视频
  2. 标记需要替换或调整的分镜（如分镜03、分镜07画面不稳定）
  3. 对于标记的问题分镜：
     - 方案A：重新调整分镜视频提示词，再次调用视频生成 API
     - 方案B：使用专业剪辑软件对该片段进行稳定/调色处理
     - 方案C：手动创建该分镜的替代画面（实拍/AE动画/素材库）

手动剪辑步骤：
  第一步：导入素材
    将 output/storyboards/videos/ 下所有分镜视频导入剪辑软件

  第二步：按顺序排列
    参考 output/storyboards/exports/分镜顺序清单.md 中的顺序排列

  第三步：添加转场
    参考 output/storyboards/exports/转场配置参考.md 的推荐转场类型
    在两个分镜之间添加转场效果（叠化/擦除/闪光等）

  第四步：叠加字幕
    从 output/01_初版字幕详细文稿.md 复制字幕文本
    在各分镜提示词文件中确认每条字幕的时间码
    在剪辑软件中创建字幕轨道

  第五步：音频处理
    · 添加背景音乐（BGM）
    · 录制或导入配音旁白（VO）
    · 调整音量平衡

  第六步：色彩校正
    · 统一全片色温、对比度、饱和度
    · 对有问题的分镜单独调色

  第七步：导出成片
    · 格式：MP4 (H.264)
    · 分辨率：1920×1080
    · 帧率：24fps
    · 比特率：≥ 15Mbps
```

### 分镜替换指南

如果某个分镜需要重新生成：

1. **仅调整提示词重新生成该分镜**：
   - 修改对应分镜文件中的「分镜视频提示词」
   - 使用关键帧图片 + 新提示词重新调用 Agnes Video
   - 将新生成的视频替换 `videos/` 中的对应文件

2. **生成多版本备选**：
   - 对同一分镜使用略有不同的提示词生成 2-3 个版本
   - 保存为 `分镜XX_视频_v1.mp4`、`分镜XX_视频_v2.mp4`
   - 在剪辑时选择最佳版本

3. **关键帧不满意时**：
   - 调整「分镜图提示词」重新生成关键帧图片
   - 使用新关键帧 + 视频提示词重新生成视频

## 质量检查清单

### 内容维度
- [ ] 字幕是否完整覆盖核心信息点（痛点、方案、功能、价值、案例）？
- [ ] 信息逻辑链条是否清晰（为什么需要→怎么解决→效果如何→谁在用）？

- [ ] 转场设计是否流畅自然？
- [ ] 关键帧图片是否能有效支撑视频生成？

### 技术维度
- [ ] 所有提示词语言选择是否合适（中文/英文）？
- [ ] 视频参数是否符合 Agnes API 要求（num_frames = 8n+1）？
- [ ] 分镜时长总和是否接近目标总时长？
- [ ] 字幕条数是否符合3-4秒/条的节奏？

## 注意事项

1. **⚠️ 必须先检查配置**：每次调用本 Skill 时，第零阶段的前置配置检查为强制步骤，不得跳过。未通过配置检查则立即终止，提示用户完成配置。
2. **⚠️ 必须完成需求四问**：第一阶段的任务需求、目的、背景、参考文档四个维度必须逐一确认，不可跳过或简化。
3. **🔗 HyperFrames 是拼接必需品**：多分镜视频的合成拼接依赖外部 HyperFrames Skill，在第零阶段即需检查。若不安装，视频将无法自动合成。
4. **🔄 生成阶段必须并行**：第五、六阶段的图片和视频生成，务必使用 Subagent 并行执行，不得逐个串行生成。分组建议每组3-5个分镜。
5. **💾 全量保留中间产物**：所有分镜图片、分镜视频、提示词文件、设计文档均完整保留，不得删除。若 AI 生成效果不理想，用户可基于全部中间文件自行手动剪辑。
6. **必须先收集素材**：在开始创作前，务必询问用户是否有参考文件和素材
7. **保持风格统一**：所有分镜的视觉风格、色调、质感应保持一致
8. **提示词择优选择语言**：中文场景默认使用中文提示词，如AI工具对英文响应更优则切换英文
9. **图片→视频的依赖**：视频生成依赖关键帧图片质量，确保图片生成满意后再进行视频生成
10. **分镜衔接关注**：相邻分镜的画面应有合理的视觉过渡逻辑，转场类型需在分镜设计阶段预先规划
11. **字幕可读性**：每条字幕8-18字，确保观众在3-4秒内能读完
12. **信息密度适中**：避免单条字幕承载过多信息，一句一意
