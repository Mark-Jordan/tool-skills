# 提示词质量体系

## 目标
把“资料理解”转换成“可稳定生成画面”的提示词，而不是把文案改写成说明句。

## 生成链路
1. 资料分析
2. 创作理解
3. 分镜意图
4. `prompt_brief`
5. `image_prompt`
6. `video_prompt`
7. `negative_prompt`
8. 自动质检

## `prompt_brief`
每个分镜先写 brief，再写最终提示词。

```yaml
visual_goal: ""
concrete_subjects: []
scene_type: ""
camera:
  shot_size: ""
  angle: ""
  lens: ""
motion_intent: ""
tone: ""
must_avoid: []
```

## 图片提示词规则
- 只描述一个核心画面。
- 使用具体名词，不用空泛口号。
- 写清主体、环境、构图、光线、材质、色调、镜头。
- 画面内文字默认禁止。
- 需要留字幕区时，直接写“底部留白”。

推荐顺序：
`主体 -> 环境 -> 动作/状态 -> 构图 -> 光影 -> 风格 -> 质量约束 -> 禁止项`

## 视频提示词规则
- 只描述从关键帧开始的连续运动。
- 只保留一个主动作和一个主镜头运动。
- 写清起始状态、变化过程、结尾状态。
- 不要在视频提示词里要求生成字幕文字。
- 不要引入新场景和新人物。

推荐顺序：
`起始帧 -> 主体运动 -> 镜头运动 -> 画面变化 -> 节奏 -> 结尾状态 -> 禁止项`

## 语言选择
- 默认使用与素材一致的语言。
- 如果英文更适合当前模型，就改用英文。
- 以“画面准确”优先，不以语言偏好优先。

## 自动质检
每条提示词必须检查：
- 是否只有一个核心场景
- 是否只有一个核心主体
- 是否能直接转成画面
- 是否避免抽象营销词堆叠
- 是否明确禁用画面文字
- 视频提示词是否只写运动，不重写文案

不通过就重写，不进入生成阶段。
